한문 텍스트 교감기 개선 사항
============================

1. 파일명 불일치 해결
   - run.py가 index.html, 실행.html 등 여러 파일명을 시도하도록 개선
   - 파일을 찾지 못할 때 더 명확한 오류 메시지 제공

2. 에러 처리 강화
   - 파일 읽기 실패 시 명확한 오류 메시지
   - 인코딩 오류 시 사용자에게 알림
   - 빈 파일 처리 및 경고
   - 비교 중 오류 발생 시 부분 결과라도 표시

3. 사용자 피드백 개선
   - 파일 상태 표시에 전체 문자 수와 한자 수 표시
   - 한자가 없는 파일에 대한 경고
   - 비교 중 오류 발생 시 상세한 오류 메시지

4. 안정성 향상
   - try-catch 블록 추가로 예외 상황 처리
   - GUI 백엔드 로드 실패 시 대체 방법 시도
   - 빈 텍스트나 한자 없는 텍스트 처리

5. 사용성 개선
   - 파일 삭제 시 상태 초기화
   - 오류 발생 시 입력 필드 초기화
   - 더 명확한 상태 메시지

6. 핵심 버그 수정: non-CJK 문자 위치 복원 문제 해결
   - 문제: 표점, 문장부호, 영어, 숫자 등이 원본 위치가 아닌 잘못된 위치에 표시됨
   - 원인: buildPairHtml 함수에서 segments를 사용하여 순차적으로 인덱스를 증가시키면서,
           LCS 비교 후 변경된 순서와 원본 텍스트의 인덱스가 맞지 않았음
   - 해결: 
     * separateCJK 함수에서 non-CJK 문자를 배열이 아닌 문자열로 저장하도록 수정
     * compareTexts 함수에서 alignment 정보를 반환값에 포함
     * buildPairHtml 함수를 완전히 재작성하여 segments 대신 alignment를 직접 사용
     * alignment의 원본 인덱스(idxA, idxB)를 사용하여 non-CJK 문자를 정확한 위치에 복원
   - 결과: 이제 표점, 문장부호, 영어, 숫자 등이 원본 파일의 정확한 위치에 표시됨
